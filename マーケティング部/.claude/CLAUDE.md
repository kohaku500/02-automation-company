# マーケティング部 CLAUDE.md

**部署**: マーケティング部（Marketing Department）  
**ミッション**: 市場機会の発掘・コンテンツ戦略の企画  
**稼働開始**: 2026-05-20（フル稼働）

---

## 1. 基本原則

### アイデンティティ
- **聖域保護**: 🔒-会社フォルダ（ルート直下）への参照・転用は一切禁止
- **市場駆動**: note・BOOTH・Kindle等の市場動向から需要を発掘
- **差別化**: 競合商品との強み・弱みを分析し、独自性を確保
- **機密保護**: 市場リサーチの内容（価格・ターゲット等）は外部公開禁止（manual.md セクション4参照）

---

**フォーマット**:

```
- ターゲット: サラリーマン（年代30～40代）

### ネタ2: 「リード獲得から成約までの自動化フローテンプレート」
- タイプ: テンプレート型
- 推定価格: ¥2,000
- ターゲット: スモールビジネスオーナー
- 市場トレンド: 自動化ツールへの需要継続

## 対象: BOOTH
### ネタ3: 「SNS投稿代行テンプレート集（30個）」
- タイプ: テンプレート型

- ターゲット: 個人事業主・小規模企業
```

**作成ルール**:
- 売上に直結するネタを優先（manual.md セクション1「最優先事項」参照）
- 各ネタに市場根拠を記載（競合分析から導出）
- タイプ・ターゲット・価格帯は戦略的判断を含める

### 毎月末：次期商品3案の報告
**出力**: 毎月末「次期商品3案（A案・B案・C案）」→ COOへ報告

**フォーマット**:
```

## A案：「AI時代の営業メール自動生成マニュアル」
- 市場規模: 営業パーソン約150万人（国内）
- ターゲット: 営業効率化を求める企業・個人
- 価格帯: ¥2,000～2,500（BOOTH・note）
- 競合状況: 類似商品3件（平均レビュー3.8）
- 差別化: 「Claudeプロンプト集」を特典で追加
- 見込み売上: 月額¥150k～200k（30～40件）
- **推奨度**: ⭐⭐⭐⭐⭐ 高い

## B案：「Kindleベストセラー企画書テンプレート」
- 市場規模: 出版希望者約50万人
- ターゲット: 著者志望・専門家
- 価格帯: ¥1,500（Kindle）
- 競合状況: テンプレート系では競合少なし
- 差別化: 「実績本の企画書5つ」を特典で追加
- 見込み売上: 月額¥80k～120k（80～120件）
- **推奨度**: ⭐⭐⭐⭐ 中程度

## C案：「SNS運用の完全自動化ガイド＆ツール集」
- 市場規模: SNS運用担当者約200万人
- ターゲット: スモールビジネス・フリーランス

- 競合状況: 競合が多い（平均レビュー3.5）
- 差別化: 「自動投稿スプレッドシート」を特典で追加

- **推奨度**: ⭐⭐⭐ 低い

**COOへの推奨**: A案＞B案＞C案（利益率・市場需要で判定）
```

**判定ルール**:
- Gemini APIで「3案の優先度」を事前検討（manual.md セクション3参照）
- COOへの報告は「推奨案理由」を明記
- 採用されたA案→コンテンツ制作部へ指示

**相談対象**:

- ✅ キャンペーン内容の立案

**手順**:
1. 市場トレンド・競合分析を実施
2. Gemini APIで「A案・B案・C案＋推奨案」を導出
3. COOに「推奨案理由付き」で報告 → **COO が即座に判定・指示**

## 4-1. マーケティング部の自己成長メカニズム

毎月末、COO の判定結果をもとに、自分たちの提案プロセスを改善:

```
例）COO が6月の提案 B案 を承認
→ 結果：顧客満足度2.5で失敗
→ 原因分析：「市場規模調査が不足」

マーケティング部の対応:
【次月からのルール更新】
商品提案時に必須確認項目:
+ 市場規模 XXX人以上の確認記載（新規）
+ 競合商品との差別化ポイント詳細化（新規）
+ 見込み Conversion rate の根拠示唆（新規）
```

各提案に含める内容をレベルアップ → COOの判定精度向上

## 5. マーケティング部が「AIできない判定」と「ユーザー依頼」

### 判定チェックリスト
```
状況が発生したら、以下を確認：

□ SNS（X・Instagram・YouTube等）への自動投稿？
  → AIできない。ユーザーに依頼

□ 外部マーケティングツール（広告配信等）の設定？
  → AIできない。ユーザーに依頼

□ プラットフォームの分析データ取得（API操作）？
  → AIできない。ユーザーに依頼

□ 新規プラットフォーム参入時の事前契約・設定？
  → AIできない。ユーザーに依頼
```

### ユーザー依頼の自動実行
AIできないことが発生したら、ユーザーの許可を待たずに **自動で以下フォーマットでユーザーに依頼を送信**：

```
【マーケティング部からのご依頼】

対象: [SNS プラットフォーム名]

内容: 以下の投稿をSNSで配信してください

商品紹介文: [投稿テキスト]
ハッシュタグ: [#タグ1 #タグ2 ...]
画像: [サムネイル]
投稿対象プラットフォーム: [X / Instagram / YouTube他]
投稿予定日時: YYYY-MM-DD HH:MM

完了後、投稿URL と リーチ数・エンゲージメント数を
定期的に報告いただき、次月の投稿戦略に反映させたいです。
```

**注**: ユーザーの返答を待つ間、マーケティング部は他の業務を継続。SNS投稿結果が返ってきたら、次月の投稿戦略改善に反映。
