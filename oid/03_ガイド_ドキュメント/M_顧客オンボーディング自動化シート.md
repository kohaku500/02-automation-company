## M-4. 進捗トラッキングシート（内部管理用）

### Google Sheets "Onboarding-Tracker"

#### Column 定義

```
A: 顧客ID
B: 企業名
C: 成約日（Day 0）
D: Day 0 ウェルカムメール送信完了
E: Day 1 ガイダンス動画確認（Y/N）
F: Day 2 説明会参加（Y/N）
G: Day 8 初期成果報告メール送信完了
H: Day 15 チャーン防止メール送信（必要に応じて）
I: Day 25 30日レビューメール送信完了
J: オンボーディング完了度（%）
K: チャーン防止対象（Y/N）
L: 継続意向（高/中/低）
M: アップセル推奨度（高/中/低）
N: 備考
```

#### 自動計算式の例

**Column J（完了度）**:
```
=COUNTIFS($D$2:$D$1000, TRUE, $E$2:$E$1000, "Y", $F$2:$F$1000, "Y", 等) / 7 * 100
```

**Column K（チャーン防止対象）**:
```
=IF(AND(E=FALSE, F=FALSE), "Y", "N")
→ ガイダンス & 説明会どちらも未実施の顧客をフラグ
```

**Column L（継続意向）**:
```
=IF(J < 40%, "低", IF(J < 70%, "中", "高"))
→ 完了度に基づいて自動判定
```

## M-5. チャーン防止ロジック

### チャーン検知ロジック

**リスク指標** (以下のいずれかに該当):

```
1. 初期セットアップ非完了（Day 3 時点）
   → 条件: Day 2 説明会に未参加 & データ入力 0 件
   → アクション: "セットアップサポート" メール送信

2. 利用頻度低下（Day 7 時点）
   → 条件: 登録日数が 1 日以下（週内）
   → アクション: "チャーン防止メール" 送信 + 電話フォロー予約案内

3. サポート要望多発（Day 15 時点）
   → 条件: サポートメール / チャット 5 件以上
   → アクション: 専任サポーター割り当て

4. 評価低下（Day 25 時点）
   → 条件: NPS < 0（デトラクター判定）
   → アクション: "カスタマーサクセス面談" 予約案内 + 返金保証説明
```

### 自動対応フロー

```
チャーン検知
    ↓
Google Sheets で自動フラグ（Column K = "Y"）
    ↓
Google Apps Script トリガー実行
    ↓
専用メールテンプレート選択 & 自動送信
    ↓
対応内容を "Churn-Log" sheet に記録
    ↓
営業 / サポートへの Slack アラート
    ↓
フォローアップ完了まで監視継続
```

## M-6. Google Apps Script 実装

### スクリプト 1: 自動メール送信（トリガー: 毎日 09:00）

```javascript
function sendOnboardingEmails() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Onboarding-Tracker");
  const data = sheet.getDataRange().getValues();
  
  // ヘッダーをスキップして、各行を処理
  for (let i = 1; i < data.length; i++) {
    const customerId = data[i][0];
    const email = data[i][1]; // 企業名から後で email を取得
    const startDate = new Date(data[i][2]); // Day 0（成約日）
    const today = new Date();
    const daysPassed = Math.floor((today - startDate) / (1000 * 60 * 60 * 24));
    
    // Day ごとのメール送信ロジック
    if (daysPassed === 0) {
      sendEmail(email, "ウェルカムメール"); // Template 1
      sheet.getRange(i + 1, 4).setValue(true); // Day 0 ✓
    } else if (daysPassed === 1) {
      sendEmail(email, "説明会リマインダー"); // Template 2
    } else if (daysPassed === 8) {
      sendEmail(email, "初期成果報告"); // Template 3
    } else if (daysPassed === 15 && shouldSendChurnEmail(sheet, i)) {
      sendEmail(email, "チャーン防止"); // Template 4
    } else if (daysPassed === 25) {
      sendEmail(email, "30日レビュー"); // Template 5
    }
  }
}

function sendEmail(email, templateType) {
  // メールテンプレートの取得
  const template = getEmailTemplate(templateType);
  
  // メール送信
  GmailApp.sendEmail(email, template.subject, template.body);
  
  // ログ記録
  logEmailSent(email, templateType);
}

function shouldSendChurnEmail(sheet, rowIndex) {
  // チャーン検知ロジック
  const setupCompleted = sheet.getRange(rowIndex + 1, 6).getValue();
  const dataEntered = sheet.getRange(rowIndex + 1, 7).getValue();
  
  return setupCompleted === false && dataEntered < 10;
}
```

### スクリプト 2: チャーン検知アラート（トリガー: 毎日 08:00）

```javascript
function detectChurnRisks() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Onboarding-Tracker");
  const data = sheet.getDataRange().getValues();
  
  const churnCustomers = [];
  
  for (let i = 1; i < data.length; i++) {
    const customerId = data[i][0];
    const completionRate = data[i][9]; // Column J
    
    // チャーン対象（完了度 < 40%）
    if (completionRate < 40 && i > 7) { // Day 7 以降
      churnCustomers.push({
        customerId: customerId,
        completionRate: completionRate
      });
      
      // Slack にアラート
      sendSlackAlert(customerId + " が チャーン リスク detected");
    }
  }
}
```

## M-7. 月別チャーン率・継続率の追跡

### 継続率トラッキングシート（内部管理用）

**Sheet 名**: "Retention-Analytics"

#### Column 定義

```
A: 月（例: 2026-06）
B: 当月成約数
C: 当月オンボーディング完了数（Day 30 時点）
D: 当月継続数（Day 31 継続契約確定）
E: 継続率 (%) = D / B * 100
F: チャーン数 = B - D
G: チャーン率 (%) = F / B * 100
H: 平均NPS
I: 主なチャーン理由（テキスト）
J: 改善施策（テキスト）
```


## M-8. オンボーディング成功の KPI

### 目標設定

| KPI | Day 7 目標 | Day 14 目標 | Day 30 目標 |
|---|---|---|---|
| **オンボーディング完了率** | 60% | 85% | 95% |
| **初期利用開始率** | 80% | 95% | 100% |
| **サポートチケット数** | < 5件 | < 3件 | < 1件 |
| **NPS（Net Promoter Score）** | N/A | N/A | > 40 |
| **継続率（Day 30）** | N/A | N/A | > 85% |
| **アップセル準備率** | N/A | N/A | > 30% |

### 失敗パターン時の対応

**パターン 1**: Day 7 完了率 < 40%
→ 原因: セットアップの複雑性 or サポート不足
→ 対応: 動画ガイド再制作 + サポーター増員 + Day 5 個別面談追加

**パターン 2**: NPS < 20（デトラクター多発）
→ 原因: 製品不具合 or 期待値とのズレ
→ 対応: 顧客フィードバック会議 + 製品改善優先順位付け + 返金保証適用判定

**パターン 3**: チャーン率 > 50%
→ 原因: 根本的なニーズ / プロダクト不適合
→ 対応: 営業ターゲット見直し + ICP（Ideal Customer Profile）再定義
