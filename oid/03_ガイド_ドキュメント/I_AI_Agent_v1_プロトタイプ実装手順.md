# I. AI Agent v1 プロトタイプ実装手順

**制作日**: 2026-05-03  
**概要**: 初期段階で稼働させる自動応答・情報収集・顧客対応の3つのAIエージェントの構築手順。Day 1 から Day 7 で最小機能セット(MVP)を完成させる。

---

## 目次

1. Agent 1: 自動営業応答エージェント（初期化・テスト・運用開始）
2. Agent 2: リード情報取得エージェント（顧客データ自動化）
3. Agent 3: 顧客対応サマリーエージェント（メール・チャット自動整理）
4. 統合テストフロー
5. 本番環境への移行

---

## I-1. Agent 1: 自動営業応答エージェント

### 目的
営業メール返信に自動で応答し、ファネル最上流の反応確認と初期顧客分類を行う。

### 機能要件
- 顧客メールを受け取る（Gmail または Webhook 経由）
- 事前設定したテンプレートベースで自動応答を生成
- 顧客の対応ステータス（関心高 / 中 / 低）を判定
- 担当者への通知メールを自動送信

### Day 1-2: 初期化フェーズ（推定時間: 120分）

#### Step 1-1: 基盤システムの選定（30分）
**選択肢**:
- **Option A**: Google Apps Script + Gmail + Spreadsheet（推奨・無料・初期段階向け）
  - メリット: セットアップ簡単、Google 連携優位、コスト0円
  - デメリット: 処理速度や複雑な条件分岐は限界あり
  
- **Option B**: Make（旧 Integromat）+ 外部 AI API（Claude / OpenAI）
  - メリット: 視覚的設計、複数サービス連携可能
  - デメリット: 月額 $10-20 から発生
  
- **Option C**: Zapier + 外部 AI API
  - メリット: 既成インテグレーション豊富、UI わかりやすい
  - デメリット: 料金体系が複雑

**推奨**: **Option A**（Google Apps Script）から開始。以降の成長段階で Option B / C に移行。

#### Step 1-2: Gmail 設定（30分）
1. **Gmail 自動フィルタ設定**
   - 営業返信の専用ラベル作成（例: "Agent-Inbox"）
   - 条件: 件名に特定キーワード含む、または特定アドレスから来たメール
   - アクション: 自動的に "Agent-Inbox" ラベルを付与

2. **API 認証設定**
   - Google Cloud Console で新規プロジェクト作成
   - Gmail API を有効化
   - OAuth 2.0 認証情報（クライアント ID / シークレット）を取得

#### Step 1-3: 自動応答テンプレート定義（30分）
**テンプレート A**: 関心度判定テンプレート（応答パターン別）

| 顧客メールの特徴 | 判定 | 自動応答テンプレート |
|---|---|---|
| 具体的な質問・ユースケース記載 | **関心度: 高** | 「貴社の○○のニーズに対応した詳細プランのご提案です...」 |
| 簡潔な問い合わせ | **関心度: 中** | 「ご関心ありがとうございます。初回相談時間の予約リンク...」 |
| 自動返信のみ / テンプレート的 | **関心度: 低** | 「情報請求のみの場合...」 |

**Google Spreadsheet で保管**: "Agent-Templates" という専用シート
- Column A: パターン識別キーワード
- Column B: 判定ステータス
- Column C: 自動応答テンプレート本文
- Column D: 先行フォローアップ日

#### Step 1-4: Google Apps Script コード実装（30分）

```javascript
// Google Apps Script: Gmail 自動応答ボット（概要）

function checkAndReplyToEmails() {
  // 1. "Agent-Inbox" ラベルから未処理メールを取得
  const gmailLabel = GmailApp.getUserLabelByName("Agent-Inbox");
  const threads = gmailLabel.getUnreadThreads();
  
  // 2. 各メールを処理
  threads.forEach((thread) => {
    const messages = thread.getMessages();
    const latestMsg = messages[messages.length - 1];
    
    // 3. メール本文から顧客の関心度判定
    const interestLevel = classifyInterest(latestMsg.getPlainBody());
    
    // 4. テンプレートから応答メール取得
    const responseTemplate = getTemplateByLevel(interestLevel);
    
    // 5. 自動応答メール送信
    latestMsg.reply(responseTemplate);
    
    // 6. Spreadsheet に記録
    logToSheet(thread, interestLevel);
  });
}

function classifyInterest(emailBody) {
  // 簡易版: キーワード検索で判定
  if (/具体的な質問|ユースケース|相談したい/.test(emailBody)) {
    return "高";
  } else if (/ご関心|問い合わせ|詳細/.test(emailBody)) {
    return "中";
  }
  return "低";
}

function getTemplateByLevel(level) {
  const sheet = SpreadsheetApp.getActiveSheet();
  // テンプレート Spreadsheet から該当行を取得・返却
  // （実装: VLOOKUP 相当の処理）
}

function logToSheet(thread, level) {
  const sheet = SpreadsheetApp.getActiveSheet();
  sheet.appendRow([
    new Date(),
    thread.getFirstMessageSubject(),
    thread.getMessages()[0].getFrom(),
    level,
    "未対応",
  ]);
}
```

**備考**: 実装は **Google Apps Script の標準ライブラリ** を使用。複雑な AI 判定は後段で Gemini API に外部化予定。

### Day 3-4: テスト・改善フェーズ（推定時間: 90分）

#### Step 1-5: ローカルテスト実行（30分）
1. **テスト用メールアカウント作成**: test-customer-01@gmail.com など 3～5 個
2. **各パターンのテストメール送信**:
   - "具体的な質問"パターン → "関心度: 高" で判定されるか
   - "簡潔な問い合わせ"パターン → "関心度: 中" で判定されるか
   - 無関係なメール → 処理スキップされるか
3. **応答メール確認**: テンプレートが正確に返送されているか

#### Step 1-6: パフォーマンス測定（30分）
- **メール処理遅延**: 受信から応答までの時間（目標: < 5 分）
- **判定精度**: 関心度判定の正確性（目標: 80% 以上）
- **エラーログ**: エラーが生じた場合の記録と原因特定

#### Step 1-7: テンプレート最適化（30分）
- テスト結果から応答テンプレートを微調整
- "関心度: 中" の顧客から返信率が低い場合 → 本文の CTA（行動喚起）を強化
- 自動応答であることが顧客に即座に伝わるよう署名を修正

## I-2. Agent 2: リード情報取得エージェント

### 目的
顧客からの問い合わせ内容から自動的に企業情報・ニーズを抽出し、CRM システムに登録する。

### 機能要件
- メール / フォーム経由の顧客情報を自動抽出
- 企業規模・業種・ニーズを分類
- 重複排除（既登録顧客の判定）
- Google Sheet または CRM に自動ロード

### Day 1-3: 初期化フェーズ（推定時間: 90分）

#### Step 2-1: データスキーマ定義（30分）
**Google Sheet "CRM-Master"** の列定義:

| Column | 説明 | 型 | 必須 |
|---|---|---|---|
| A | 顧客ID（自動採番） | 数値 | ○ |
| B | 企業名 | テキスト | ○ |
| C | 代表メール | メール | ○ |
| D | 企業規模（従業員数推定） | カテゴリ (1-10 / 11-50 / 51-200 / 200+) | △ |
| E | 主要ニーズ | テキスト | △ |
| F | 初回接触日 | 日付 | ○ |
| G | ステータス | カテゴリ (新規 / 検討中 / 提案待ち / 成約) | ○ |
| H | 最終更新日 | 日付 | ○ |

#### Step 2-3: 重複検知ロジック（30分）

## I-3. Agent 3: 顧客対応サマリーエージェント

### 目的
顧客とのメール / チャット / 通話記録から重要な内容を自動抽出・サマリーし、営業進捗を可視化する。

### 機能要件
- 複数チャネル（Gmail, Slack 等）の対話記録を集約
- キー情報（要望・懸念・決定予定日）を自動抽出
- 営業担当者向けの「今日のアクション」を自動生成
- 週次進捗レポートの自動作成

### Day 1-4: 初期化フェーズ（推定時間: 120分）

#### Step 3-1: データ集約フロー設計（30分）
- **Gmail**: "Customer-Conversation" ラベルのメール
- **Slack**: 特定の DM チャネル（例: #customer-feedback）
- **外部**: Web フォーム の問い合わせ（後段で JSON として自動連携）

**データフロー**:
```
Gmail / Slack / Web Form 
  → Google Sheets "Raw-Conversations"
    → AI 処理（キーワード抽出）
      → Google Sheets "Summary-Dashboard"
        → 営業チームへの Email / Slack 通知
```

#### Step 3-2: キーワード抽出テンプレート（30分）
**Google Sheets "Extraction-Rules"** の設定:

| キーワード | 優先度 | アクション |
|---|---|---|
| "導入決定" / "契約予定" | 🔴 **高** | 営業担当者に即時通知 + CRM ステータス更新 |
| "見積もり希望" | 🟠 **中** | 営業担当者に翌日通知 + 提案フロー開始 |
| "懸念点" / "問題" | 🟠 **中** | 営業担当者にまとめて報告 |
| "他社比較" / "検討中" | 🟡 **低** | 週間レポートに含める |

#### Step 3-3: サマリー自動生成フロー（30分）
```javascript

  // 1. 過去 N 日間の会話を収集
  // 2. 時系列でソート
  // 3. キーワード抽出
  // 4. テンプレートに当てはめて サマリー生成
  // 5. CRM に記録 + Slack に通知

```

**テンプレート例**:
```

- 最終更新: 2026-05-03
- ステータス: 検討中

  ? 質問: 他社導入事例との比較情報

```

#### Step 3-4: 通知ロジック実装（30分）
- **高優先度**: Slack の営業チャネルで即時アラート
- **中優先度**: 営業担当者へ 1 日 1 回の日報メール
- **低優先度**: 週 1 回の総括レポート
