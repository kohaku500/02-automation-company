## 1. プロダクト概要

### 1.1 目的
Gmail に AI Agent を統合し、受信メールを自動分析・スコアリングして、営業チームの効率を 5～10 倍に向上させるツール。

### 1.2 主要機能
1. **メール自動分析** - 受信メール内容から購買意欲を自動判定
2. **スコアリング** - 0～100 点で見込み客の購買可能性を評価
3. **自動振り分け** - 見込み客の高さに応じて自動返信 or 営業に通知
4. **個人文体学習** - 営業メンバーの文体を学習して自動返信を生成
5. **ダッシュボード** - 全受信メール、スコア、処理状況を可視化
6. **CRM 連携** - Google Sheets に自動記録
7. **権限管理** - 管理職は全体、営業は個人データのみ表示

## 6. 機能仕様

### 6.1 メール自動分析

**入力：** メール本文、送信者情報  
**処理：** Claude API による自然言語処理  
**出力：** スコア、購買シグナル、企業情報  

**分析項目：**
- 予算有無（「予算承認済み」「検討中」「予算未決定」）
- 購入時期（「来月」「今月」「3ヶ月以内」）
- 決定権（「決定権者」「稟議中」「相談中」）
- 競争状況（「単独検討」「複数社比較」「ベンダー決定済み」）
- メール返信の熱度（返信速度、用語使用、具体性）

### 6.2 スコアリングロジック

```
基本スコア = 50点

加点要素：
+ 10点 ：「来月導入」「今月導入」など時間軸が明確
+ 10点 ：「予算承認済み」など予算有無が確定
+ 10点 ：決定権者が送信
+ 5点  ：複数回メール交換（リエンゲージメント）
+ 3点  ：返信が 24 時間以内

減点要素：
- 10点 ：「参考までに情報を」など関心度が低い
- 10点 ：「複数社検討中」など競争が激しい
- 5点  ：「いずれ検討」など時間軸が曖昧
- 3点  ：返信が 1 週間以上

最終スコア：基本スコア + 加点 - 減点（0～100）
```

### 6.3 個人文体学習

**初期セットアップ：**
1. 営業メンバーが過去のメール 10～20 件を提供
2. AI が文体パターンを学習
   - フォーマルさレベル（フォーマル～カジュアル）
   - 敬語の使用程度
   - 文の長さ傾向
   - よく使う表現
   - 署名スタイル

**自動返信生成：**
学習した文体で、メール内容に応じて自動返信を生成
```
例：田中太郎（フォーマル志向）の返信
「平素よりお世話になっております。
メールをありがとうございました。
〇〇につきましては、弊社でサポート可能です。
詳細は別途打ち合わせにてご説明させていただきたく存じます。」

例：鈴木花子（カジュアル志向）の返信
「こんにちは！ご連絡ありがとうございます。
〇〇についてですが、対応できますよ。
来週のスケジュールで詳しくご説明させていただきます。」
```

### 6.4 権限管理

| 権限レベル | 見えるデータ | 操作可能 |
|-----------|----------|--------|
| **管理職** | 全営業の全データ（メール、スコア、処理履歴） | ダッシュボード確認、レポート生成 |
| **営業メンバー** | 自分のメール詳細 + 他営業の勤怠情報のみ | メール転送、自動返信実行 |

## 7. 技術仕様

### 7.1 技術スタック

| レイヤー | 技術 |
|--------|------|
| **フロントエンド** | Gmail Add-On UI（Google Apps Script） |
| **バックエンド** | Google Apps Script |
| **AI 処理** | Claude API |
| **データ保存** | Google Sheets（CRM） |
| **認証** | Google OAuth 2.0 |
| **メール処理** | Gmail API |
| **通知** | Slack API（オプション） |

### 7.2 Google Apps Script ファイル構成

```
📁 AI Agent Sales Assistant Project
├── 📄 Main.gs              【メイン実行関数】
├── 📄 EmailHandler.gs      【メール受信・処理】
├── 📄 AIAnalyzer.gs        【AI 分析・スコアリング】
├── 📄 StyleLearner.gs      【個人文体学習】
├── 📄 AutoResponder.gs     【自動返信生成・送信】
├── 📄 CRMLogger.gs         【Google Sheets 記録】
├── 📄 Dashboard.gs         【ダッシュボード表示】
├── 📄 PermissionManager.gs 【権限管理】
└── 📄 Config.gs            【設定・定数】
```

### 7.3 API キー・認証

```
【必須設定】
- Claude API キー（企業が取得）
- Google OAuth スコープ（Gmail, Google Sheets）

【安全性】
- API キーは Google Apps Script の PropertiesService に暗号化保存
- ユーザーが管理者画面で設定
```

### 7.4 定期実行トリガー

```
関数：checkAndProcessEmails()
頻度：毎時間（毎正時）
処理内容：
- tanaka.agent@abc.com 等の未処理メールをチェック
- AI が分析・スコアリング
- 必要に応じて本人メールに転送または自動返信
- 結果を Google Sheets に記録
```

### 8.1 ユーザーインストール手順

```
【ステップ 1】Google Workspace Marketplace で検索
```

## 10. セキュリティ・コンプライアンス

### 10.1 データ保護
- Google OAuth による安全な認証
- API キーは暗号化保存
- Google Sheets はユーザーのドライブに保存（Google の保護対象）
- メール内容はメモリに一時保存、処理後削除

### 10.2 権限・アクセス制御
- 営業メンバーは自分のデータのみ表示
- 管理職も組織内のデータのみ表示
- 外部ユーザーはアクセス不可

### 10.3 監査ログ
```
処理履歴を Google Sheets に記録：
- メール受信日時
- AI スコア
- アクション（転送 or 自動返信）
- 実行者
```

## 12. 付録

### 12.1 用語集

- **AI Agent** : Claude API を使った自動分析エージェント
- **スコアリング** : 購買意欲を数値化（0～100 点）
- **購買シグナル** : メール内に含まれる購買の可能性を示す情報
- **文体学習** : ユーザーのメール文体パターンを AI が学習
- **Add-On** : Gmail に統合される拡張機能

### 12.2 FAQ

**Q: セットアップに技術知識は必要ですか？**  
A: 不要です。Gmail の Add-On をインストール → Claude API キーを入力 → 運用開始。非技術者でも可能です。

**Q: 既存メールシステムと連携できますか？**  
A: Gmail を使っていれば OK。Outlook 等他のメールシステムは今後対応予定です。

**Q: メール内容は保存されますか？**  
A: スコアと企業情報は Google Sheets に記録されますが、メール本文そのものは保存されません。
