### **Slide 1: 営業の現実（問題提起）**

**タイトル**: 「営業チームの毎日」

**必須要素**:
1. **ビジュアル**: 営業デスクの朝のシーン（イラスト or 実写）
   ```
   朝9:00 → メール受信 100+ 件
   朝9:30 → 「誰が買いそうか」判定に 30 分費やす
   朝10:00 → ようやく顧客対応開始
   その間、本当に買いそうな客への返信が遅延
   ```

2. **ナレーション**（30秒）:
   ```
   「営業チームは毎日、大量のメール受信の中から
   『本当に買ってくれそうな客』を見分けるのに
   平均 30 分を費やしています。
   
   でも、その時間が長すぎる。
   本当に買いそうな客への対応が遅れ、
   競合に案件を奪われる。」
   ```

3. **数字**:
   - 受信メール: 100～200 件/日
   - 仕分け時間: 30 分/日
   - 見込み客率: 2～3%（3～5 件）
   - 月間時間浪費: 10 時間
   - **年間 120 時間 = 15 営業日分の損失**

4. **聴き手への問いかけ**:
   ```
   「もし、この 30 分を 3 分に短縮できたら？
   その 27 分を、本当に買いそうな客への対応に使ったら？」
   ```

### **Slide 2: その問題の金銭的インパクト**

**タイトル**: 「実は、¥2.4M の機会損失」

**必須要素**:
1. **計算式を見せる**（視覚化）:
   ```
   年間 120 時間の営業時間浪費
   ÷ 営業の年間労働時間 2,000 時間
   = 営業工数の 6%
   
   営業チーム 10 人 × 年間売上 ¥2M = ¥20M
   × 6% = ¥1.2M の直接的な売上損失
   
   さらに、返信遅延による失注率 10% 上乗せ
   = ¥1.2M + ¥2M × 10% = ¥2.2M～¥2.4M
   ```

2. **ナレーション**（20秒）:
   ```
   「この 30 分の時間浪費は、
   実は年間 ¥2.4M の売上機会損失につながっている。
   
   営業チーム 10 人の企業であれば、これは
   営業マン 1 人分の年間売上に相当します。」
   ```

3. **ビジュアル**: 積み上げ棒グラフ
   ```
   ¥20M ┌─────────────────┐
        │ 年間売上        │
        │ ¥2.4M 損失     │ ← ここが機会損失
   ¥17.6M└─────────────────┘
   ```

4. **深い問い**:
   ```
   「この ¥2.4M の損失を、何で補てんしているのか？」
   」→「つまり、この損失がなくなったら、
   年間 ¥2.4M の純利益が追加される」
   ```

## 📝 まとめ

このプレゼン資料定義書は、以下の原則に基づいています：

1. **ストーリー駆動** - 数字ではなく、相手の脳に刺さる物語
2. **体験させる** - 読ませるのではなく、「その営業になった気分」を体験させる
3. **金銭化** - すべての問題を「¥」に変換する
4. **クリア CTA** - 最後に「次、何をするのか」が明確

このフォーマットで再構築すれば、NotebookLM のプレゼン資料は **「伝わる」から「刺さる」に進化します。**
