## 2. note連携仕様書（パッケージ化版）

### 2-1. 概要
- **プラットフォーム**: note（記事投稿＋販売）
- **用途**: 案件1「AI時代の個人スキル販売術」の販売
- **販売価格**: ¥1,500
- **販売形式**: 有料記事（ダウンロード販売も可）

### 2-2. パッケージ化仕様

**制作部が生成・出力するセット:**
```
note_package_20260503/
├─ article_metadata.json
│  ├─ title: "AI時代の個人スキル販売術"
│  ├─ price: 1500
│  ├─ description: "営業メールを5分で自動生成..."
│  ├─ tags: ["営業", "AI", "自動化"]
│  └─ content_summary: "フリーランスの営業負担を軽減する方法"
├─ article_content.md
│  ├─ 本文（Markdown形式）
│  ├─ 3大特典の説明
│  └─ ダウンロードリンク配置
├─ thumbnail.png
│  └─ 1200×675px アイキャッチ画像
├─ bonus_pack.zip
│  ├─ lifehack: 営業メール10パターン.xlsx
│  ├─ tool: 営業フロー自動化シート.xlsx
│  └─ material: 業界別テンプレ集.pdf
└─ publication_checklist.md
   ├─ 内容チェック
   ├─ 機密情報混入チェック ✅
   └─ 投稿準備完了フラグ
```

### 2-3. 投稿手順（手動）

**オーナーが実施：**
```
Step 1: パッケージ確認
├─ publication_checklist.md で内容確認
└─ 機密情報混入なし → OK

Step 2: note管理画面で投稿
├─ 新しい記事を作成
├─ article_content.md の内容をコピー
├─ thumbnail.png をアップロード
├─ 販売ゲート設定（¥1,500）
└─ 下書き状態で保存

Step 3: プレビュー確認＋公開
├─ ブラウザで表示確認
└─ 「公開」ボタンをクリック
```

### 2-4. 投稿スケジュール

**推奨方法**: 手動投稿 + Calendar管理
```
毎週月曜 09:00 に新しい記事を投稿
（または需要に応じた柔軟な投稿）
```

**参考**: note投稿予約機能を使用可能
- 予約投稿日時を設定すれば、自動公開も可能

## 7. セキュリティ考慮事項

### 7-1. APIキー管理
- ✅ 環境変数（`.env`）にAPIキーを保存
- ✅ ファイルシステムのパーミッション制限（600）
- ✅ セキュリティスキャンで自動検出＆削除（sanitizer.py）

### 7-2. 聖域保護の確認
- ✅ 各連携スクリプトは🔒-会社フォルダにアクセスしない
- ✅ 生成コンテンツのスキャン時に機械設計情報が混入していないか確認
- ✅ プラットフォーム側に送信する前に情報システム部でチェック

### 7-3. エラーハンドリング
- ✅ API呼び出し失敗時は自動リトライ（最大3回）
- ✅ recovery_expert.py で自動修復を試行
- ✅ 修復失敗時はオーナーに通知
